# Architecture Decision Records (ADR)

Short docs for product / technical forks with trade-offs.
Format: `NNN-title.md`.

## Template

```
# NNN — Title

**Status:** proposed | accepted | superseded by NNN
**Date:** YYYY-MM-DD

## Context
Why this decision is needed.

## Decision
What we chose.

## Consequences
What follows from this — short and long term.
```

ADRs are immutable once accepted. Supersede with a new ADR rather
than editing.
